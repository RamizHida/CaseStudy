Case Study: MyFlix Movie Web App

Summary

MyFlix is a web application, built with the MERN stack, that allows its visitors to quickly skim information about movies and directors. Visitors can sign up, create a list of their favorite movies, and delete their account at any time.

Purpose

MyFlix was one of several projects from my FullStack Web Development course, CareerFoundry. MyFlix is meant to demonstrate my Javascript knowledge, from Frontend to Backend.

Goal

The goal of MyFlix was to learn how to develop a FullStack web application. The problem that I wanted to solve was to connect the client-side with the server-side to produce a fully functioning web application.

Project Details

Server-Side

By leveraging my javascript skills, I was able to use NodeJS with Express, to communicate with a non relational database (MongoDB) to develop a REST API. My Flix’s API can be accessed by way of HTTP methods, such as GET, POST, and DELETE. CRUD methods were employed to both retrieve and store data in a non relational manner. The movie data that was stored in the database was formatted in JSON, a type of file format designed for transferring data across the web. I also implemented authorization and authentication using JWT. I used Postman for testing MyFlix's API.

API Testing

Client-Side

With the completion of the server-side of MyFlix, I used React to build a Single Page Application (SPA). MyFlix’s user interface provides many page views, including: main view (which shows all available movies), a single page movie view (which shows information about the chosen movie), login view, signup view, favorite movie list view ,and account details view. By using React Bootstrap, I was able to make the user interface responsive and consistent between different page views.

What was most enjoyable?

I very much enjoyed the process of setting up and connecting to a database. During API testing with Postman, it was exciting seeing MyFlix’s API responding as intended.

What would you do differently next time?

When I build a similar application, with both a Frontend and a Backend, I would like to incorporate a relational database, such as MySQL.
